# Compose Reference Redesign

DockPilot Compose 页面需要尽量复刻用户提供的参考图：一个浅色、圆角、四栏式的 Compose 工作台，而不是当前偏监控卡片的布局。

## Scope

- 左侧项目列表使用白色毛玻璃面板，包含标题、新建项目按钮、搜索框和项目条目。
- 中间顶部显示当前项目名、路径、运行状态胶囊和紧凑工具栏。
- 主编辑区使用深色 `docker-compose.yml` 编辑器，保留现有语法高亮、滚动同步和保存行为。
- 右侧中栏显示 `AI 修复预览`，用问题卡片表达检查结果、修复前和修复后内容；已有 AI 修复结果继续可预览和应用。
- 最右侧显示项目设置表单样式，包括项目名称、项目路径、网络、重启策略、开机自启、自动更新和删除项目按钮。
- 现阶段不引入新的后端数据模型。没有持久化接口的设置项作为只读/展示型控件处理，避免破坏已有 Compose API。

## Behavior

- 项目创建、选择、编辑保存、检查配置、AI 修正、命令转 Compose、应用修正、部署、停止、重启、拉取、备份恢复继续使用现有动作。
- 顶部工具栏按钮改为参考图风格：启动、停止、重启、刷新状态、修复、保存、更多。
- 页面在没有选中项目时仍显示空白编辑提示，并禁用依赖项目路径的按钮。
- 搜索框先作为视觉和前端标记加入；如果当前页面没有搜索状态，不改变项目列表行为。
- 删除项目按钮先禁用并给出危险态视觉，避免在后端缺少删除项目 API 时误导用户执行破坏性操作。

## Visual Requirements

- 页面整体为浅色蓝灰背景、大圆角外壳、轻微内阴影和玻璃边缘。
- 项目列表条目为浅蓝选中态，未选中为白底，显示锁形/项目图标、名称、路径和右箭头。
- 编辑器接近参考图深蓝黑色面板，带顶部文件名栏、行号样式和底部状态栏。
- AI 预览区使用白色卡片，问题编号胶囊、展开符号、浅绿色修复片段。
- 项目设置区使用垂直字段，开关用蓝色/灰色滑块，删除按钮为红色描边淡底。
- 桌面端优先保持四栏比例；窄屏下降为单列/两列，不能横向溢出。

## Testing

- Smoke test 增加前端标记检查：`compose-reference-shell`、`compose-ai-issue-card`、`compose-settings-switch`、`compose-editor-statusbar`。
- `node --check web/app.js` 必须通过。
- `python3 scripts/smoke_test.py` 必须通过，证明现有 API 和前端资源标记仍可用。
- `python3 scripts/unit_test.py` 必须通过，证明后端 Compose 解析和现有逻辑未被破坏。
