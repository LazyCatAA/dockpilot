# 使用手册

本文面向实际使用者，说明 DockPilot 每个页面的功能和操作方式。

## 首次登录

第一次打开 DockPilot 会进入管理员创建页。

需要填写：

- 用户名，默认可以使用 `admin`。
- 密码，至少 8 位。

创建后会自动登录。

## 首页导航

首页导航用于保存常用服务入口，例如 NAS、下载器、媒体库、监控面板。

支持：

- 新增导航卡片。
- 编辑标题、地址、分组、短标识、颜色。
- 删除卡片。
- 按分组展示。

访问地址支持：

- `http://...`
- `https://...`
- 站内相对路径，例如 `/admin`

## 容器管理

容器管理页用于管理宿主机 Docker 容器。

顶部统计项：

- 总容器
- 运行中
- 已停止
- 有更新

这些统计项可以点击筛选容器列表。

容器卡片显示：

- 自定义图标。
- 容器名称。
- 状态圆点。
- 镜像名称。
- 运行时间或停止状态。
- 有更新时显示 `NEW` 角标。

主要操作：

- 启动或停止。
- 重启。
- 更新。
- 检查更新。
- 备份。
- 查看详情。
- 查看日志。

### 自定义容器图标

点击容器卡片左侧图标即可上传。

支持格式：

- PNG
- JPG
- WebP
- GIF

大小限制：

```text
6MB
```

### 检查更新

点击“检查更新”后，DockPilot 会执行镜像拉取并比较：

- 当前容器正在使用的镜像 ID。
- 当前标签拉取后的最新镜像 ID。

如果不同，则标记为有更新。

### 一键更新

点击“更新”后会执行：

1. 自动备份当前容器 inspect 配置。
2. 拉取最新镜像。
3. 如果是 Compose 管理的容器，优先使用 `docker compose pull` 和 `docker compose up -d --force-recreate`。
4. 如果是普通容器，尝试按原配置重建容器。

一键更新适合常规容器。复杂容器建议先备份，再手动检查 Compose 文件后部署。

### 容器备份和恢复

点击“备份”会保存：

- 容器 inspect 原始配置。
- 由容器配置生成的 `compose.yml`。

在“容器备份”区域点击“恢复为 Compose”后，会创建一个新的 Compose 项目，不会覆盖原容器。

## Compose 管理

Compose 管理页用于管理 Compose 项目。

左侧项目列表显示：

- 项目名称。
- 项目整体状态。
- 服务名称。
- 服务对应容器状态。

右侧编辑器支持：

- 编辑 `compose.yml`。
- 保存。
- 检查配置。
- 拉取镜像。
- 部署。
- 重启。
- 查看日志。
- 停止。

## 命令部署

命令部署用于把 `docker run` 命令转换成 Compose。

操作方式：

1. 进入 Compose 管理。
2. 在“命令部署”区域填写项目名称。
3. 粘贴 `docker run ...` 命令。
4. 点击“转 Compose”或“转 Compose 并部署”。

支持常见参数：

- `--name`
- `-p` / `--publish`
- `-v` / `--volume`
- `-e` / `--env`
- `--restart`
- `--network`
- `--hostname`
- `--user`
- `--workdir`
- `--entrypoint`
- `--add-host`
- `--cap-add`
- `--device`
- `--dns`
- `--privileged`
- `--init`
- `-i`
- `-t`

不常见参数会尽量跳过，转换后建议检查右侧 Compose 文件再部署。

## 文件管理

文件管理页只允许访问系统设置中配置的根目录。

支持：

- 浏览目录。
- 编辑文本文件。
- 上传文件。
- 下载文件。
- 复制。
- 移动。
- 重命名。
- 删除。
- 新建目录。

安全限制：

- 不能越过已配置根目录。
- 不能删除根目录本身。
- 大文件不会直接作为文本预览。

## 系统设置

系统设置包含：

- Docker socket 路径。
- Compose 目录。
- 文件根目录。
- 修改密码。

默认 Docker socket：

```text
/var/run/docker.sock
```

Compose 目录可以配置多个，每行一个绝对路径。

文件根目录格式：

```text
名称=/绝对路径
```

示例：

```text
files=/volume1/docker/dockpilot/data/files
docker=/volume1/docker
media=/volume1/media
```

## 桌面端和手机端

桌面端：

- 菜单固定在左侧。
- 支持隐藏/展开。

手机端：

- 自动切换为底部导航。
- 容器卡片和按钮改为单列布局。
- Compose 和文件表格会压缩为适合手机滚动的布局。
