# Home Navigation Workbench Design

DockPilot 首页导航需要从偏装饰的书签画布调整为更专业、清晰的服务入口工作台：顶部搜索框作为第一视觉层，下面按自定义分组展示书签。

## Scope

- 首页继续作为“搜索 + 分组书签”的导航页，不恢复主机状态、Docker 状态或容器监控小卡片。
- 顶部区域只保留页面标题、简短说明、网页搜索框、搜索引擎选择、搜索按钮、全局新增书签和外观设置入口。
- 书签区域按自定义分组展示，每个分组包含颜色点、分组名、数量、添加、设置和折叠操作。
- 书签卡片保持高密度白色入口样式，展示图标、标题、描述和主机名。
- 现有右键菜单继续提供内网访问、外网访问、编辑和删除。
- 现有书签编辑弹窗、分组设置弹窗、外观设置弹窗继续复用，不改变后端 API 合同。

## Non-Goals

- 不新增后端数据表或迁移。
- 不引入新的前端依赖。
- 不把首页改回资源仪表盘。
- 不改变卡片点击和右键菜单的现有行为。
- 不移除当前已经支持的卡片尺寸、样式、形状、图标上传、内外网地址和分组偏好。

## Layout

- 页面主体保留现有左侧应用导航，首页内容作为右侧大工作台面板。
- 工作台顶部使用两列布局：左侧是标题和说明，右侧是新增和设置图标按钮。
- 搜索框独立成块，桌面端宽度控制在主内容的 70-80%，内部按“图标、输入框、搜索引擎、按钮”排列。
- 书签库标题和过滤输入位于分组列表上方，过滤输入靠右，避免和网页搜索混淆。
- 每个分组独立成段，不使用重卡片包裹；用标题行和卡片网格表达层级。
- 桌面端分组卡片网格使用 `auto-fill`，优先保证同屏入口数量；移动端降为两列。

## Visual Requirements

- 整体使用浅色工作台风格：浅蓝灰背景、轻微网格纹理、白色搜索框、白色书签卡片。
- 主操作色使用克制蓝色，分组颜色只作为点状标识和卡片局部强调。
- 卡片阴影保持轻量，避免深色渐变画布压住书签内容。
- 标题字重高于正文，但不使用夸张字号；正文说明保持弱化。
- 图标按钮使用明确的图标，不用大段文字按钮占据主要视觉。
- 卡片文字必须单行省略，避免长服务名、长域名导致卡片撑开或重叠。
- 交互控件保持至少 34px 高度，主要搜索控件保持至少 48px 高度。

## Behavior

- `webSearchForm` 继续按当前搜索引擎配置打开搜索结果。
- `navSearch` 继续只过滤书签，不影响网页搜索输入。
- 分组添加按钮打开该分组下的新书签弹窗。
- 分组设置按钮打开当前分组设置弹窗。
- 分组折叠和隐藏继续写入 `/api/dashboard/nav` 偏好。
- 书签卡片左键继续优先打开 `internal_url`，没有内网地址时打开 `url`。
- 书签卡片右键菜单继续渲染在应用根层，避免被首页布局裁切。

## Data and API

- 继续使用 `/api/cards` 读取、创建、更新和删除书签。
- 继续使用 `/api/dashboard/nav` 读取和保存首页外观、搜索引擎、分组折叠、隐藏、颜色、密度和形状偏好。
- `DEFAULT_NAV_PREFS` 可调整默认文案和默认背景色，但不能删除已有字段。
- 旧数据缺少新视觉需要的字段时继续通过现有 normalize 默认值兜底。

## Responsive Requirements

- 桌面端保留左侧导航栏和右侧大工作台结构。
- 1024px 以下搜索框可压缩，但搜索输入仍优先占据最大宽度。
- 720px 以下搜索框改为纵向排列，搜索图标可隐藏。
- 手机端分组标题和操作按钮不能互相遮挡；必要时操作按钮换行。
- 手机端书签卡片保持两列，卡片高度不低于当前 smoke test 约束。
- 页面不允许横向溢出，继续纳入 `responsive-baseline` 规则。

## Testing

- `web/index.html` 资源版本递增，并同步 `scripts/smoke_test.py` 中对应断言。
- `node --check web/app.js` 必须通过。
- `python3 -m py_compile dockpilot/server.py scripts/smoke_test.py scripts/unit_test.py` 必须通过。
- `python3 scripts/smoke_test.py` 必须通过，证明首页导航标记、右键菜单、弹窗和响应式断言仍可用。
- `python3 scripts/unit_test.py` 必须通过，证明导航偏好 normalize 和后端行为没有回归。
- `git diff --check` 必须通过。
